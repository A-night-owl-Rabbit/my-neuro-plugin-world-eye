# 世界之眼 (World Eye) 插件

任务型多智能体代理路由：支持任务目标驱动、研究工作流、角色分工与按角色独立模型执行；在保留「搜索 + 执行」瘦身思路的同时，面向复杂任务做多角色编排。

## 快速开始（开箱即用）

1. 将本仓库**根目录**（即包含 `index.js`、`metadata.json` 的文件夹）放入 `live-2d/plugins/community/world-eye/`。
2. **依赖**：本仓库已附带 `node_modules`（`node-fetch` 等），一般**无需**再执行 `npm install`。若你清空过依赖，可在插件目录执行一次 `npm ci` 或 `npm install`。
3. 在 my-neuro 插件设置中配置「下级智能体」：**API Key**、**API 地址**、**模型名称**（需支持 Function Calling，例如 DeepSeek V3.2）。
4. 按需勾选「代理插件列表」与各「角色模型配置」；敏感信息请只写在本地，勿提交到 Git（见下文「隐私与安全」）。
5. 启用插件即可，**无需修改主项目核心文件**。

可选：若使用独立 `config.json`，可复制 `config.example.json` 为 `config.json` 后填写（`config.json` 已被 `.gitignore` 忽略，不会进仓库）。

## 配置项摘要

| 区域 | 说明 |
|------|------|
| 启用插件 | 是否开启世界之眼 |
| 代理插件列表 | 勾选由下级智能体实际调用的插件 |
| 下级智能体配置 | 默认执行用 LLM（API Key / 地址 / 模型 / 轮次 / 温度） |
| 角色模型配置 | 各角色可选独立模型 |
| 角色插件映射 | 各角色默认绑定的插件名（逗号分隔） |
| 任务限制 / 安全策略 | 并发与代码执行相关开关 |

## 工作原理（简述）

- 在 LLM 请求阶段拦截并调整工具列表，将大量 server-tools / MCP / 已选代理插件工具收敛为可由主模型调度的元工具与描述。
- 按任务类型与角色把执行委派给下级智能体（见 `sub-agent.js`），由其在独立对话中完成 Function Calling 循环。
- 角色级提示词与执行规则在代码中维护，用于保证多角色行为一致；**不包含**用户个人主应用系统提示词。

## 隐私与安全

- **切勿**将含真实 API Key 的 `config.json` 或带密钥的 `plugin_config` 导出文件提交到公开仓库。
- 本仓库 `.gitignore` 已忽略 `config.json`、`tool_cache.json` 等常见本地文件。
- 若你发现误提交密钥，请**立即轮换**密钥并在 Git 历史中清理敏感提交（必要时联系平台支持）。

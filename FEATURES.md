# 世界之眼 v3 — 功能说明

对应 `metadata.json` 版本 **3.0.0**，与 `index.js` / `sub-agent.js` 实现一致。

---

## 群公告版（约 10 条，可复制转发）

1. **世界之眼 v3**：把勾选「代理」的插件工具从主 AI 面前收走，换成少数总控工具，省 Token、好调度。  
2. **主 AI 四件套**：`goal` 交目标（多步优先）、`delegate` 点名一个插件办事、`research` 专题研究、`control` 查状态/取结果/排队/停止。  
3. **多步一句话说完**：画图+写文案+发小红书这类，用 **`world_eye_goal`** 整句描述，内部自动拆工作流、能并行就并行。  
4. **单插件小事**：明确只要一个插件时用 **`world_eye_delegate`**。  
5. **研究一条龙**：规划 → 搜索 → 审查 → 报告，可选「报告 + 人设化润色」（不改事实）。  
6. **同步 / 异步**：`sync` 等结果；`async` 拿任务 ID，再用 `world_eye_control` 跟进度。  
7. **下级智能体**：独立模型跑工具循环；可按「角色」换不同 API/模型（搜索/代码/生图等）。  
8. **复合模板**：内置如「小红书发布」多步流水线（需你已代理对应生图、Skills 等插件）。  
9. **安全**：代码任务可单独限并发、限插件、拦截描述里的高危命令模式。  
10. **详细表格**：见下文各节。

---

## 一、对主对话里的 AI 做了什么

| 功能 | 说明 |
|------|------|
| **工具瘦身** | 在设置中勾选「被代理」的插件后，这些插件的**原始工具**不再整包出现在主 AI 工具列表中，改由世界之眼注入少量元工具。 |
| **历史清理** | 清理对话里与已代理插件相关的旧工具调用痕迹，减轻主上下文干扰。 |

---

## 二、主 AI 可用的 4 个工具

| 工具名 | 功能 |
|--------|------|
| **`world_eye_goal`** | 提交自然语言目标（单步或**多步复合**）；**自动选工作流**；支持 sync/async；若识别为研究可带深度与输出类型（摘要 / 报告 / 报告+人设化）。 |
| **`world_eye_delegate`** | **指定插件名 + 任务描述**，由内部角色调用该插件工具；可选 `agent_role`；sync/async；可被内部判定升级为 `goal` 编排。 |
| **`world_eye_research`** | **专题研究**：规划、搜索、审查、报告；深度 quick/standard/deep；输出 summary/report/report+persona；研究用工具会过滤明显高危类工具名前缀；sync/async。 |
| **`world_eye_control`** | **status / result / raw_result**：状态与结果；**list / queue**：任务与排队；**stop / cancel**：停止或取消。 |

### `agent_role` 可选值（delegate）

`general`、`planner`、`search`、`music`、`image`、`video`、`code`、`file`、`app`、`reviewer`、`reporter`、`synthesizer`、`persona`（未指定时由插件推断）。

---

## 三、内部工作流类型（能力目录）

| 类型 | 作用 |
|------|------|
| **research_topic** | 围绕主题自动规划、搜索、审查、出报告，可转人设化表达。 |
| **image_task** | 绘画 / 生图 / 插画 / 海报等视觉类目标路由。 |
| **code_task** | 代码分析、修复、执行与结果说明（更严并发与安全限制）。 |
| **music_task** | 音乐搜索、生成、播放与状态。 |
| **file_task** | 文件读写、目录、文本整理。 |
| **app_task** | 本机应用启动、桌面侧动作。 |
| **composite_task** | 预设**复合多步**（如小红书：撰文 + 生图 + 走 Skills 发布），支持步骤依赖与并行。 |
| **planned_composite** | 无现成模板时**自动规划**并**并行**拆步执行。 |
| **auto_goal** | 由高层目标自动落入上述某一类工作流。 |

> **说明**：复合模板依赖你在「代理插件列表」中勾选并正常加载对应插件（如 `openrouter-image`、`myneuro-plugin-skills` 等），否则会报错提示缺插件。

---

## 四、下级智能体（Sub-Agent）

- 使用独立 LLM 配置（或回退主应用）发起 HTTP 请求，按轮次 **Function Calling** 直至完成或达到上限。  
- **按角色**使用不同系统提示块（规划、搜索、审查、报告、人设渲染、代码、音乐等）。  
- **共享执行规则**：优先工具取事实、禁止伪造结果、阻塞需说明原因等（见 `sub-agent.js`）。

---

## 五、插件设置页可配项（与 `plugin_config.json` 对应）

| 区域 | 功能 |
|------|------|
| **代理插件列表** | 勾选由世界之眼代管的插件；未勾选者仍直接暴露给主 AI（若该插件自身仍注册工具）。 |
| **下级智能体** | 是否使用独立模型、API Key、地址、模型名、最大轮次、温度。 |
| **角色模型配置** | 各角色可单独 API/模型/轮次/温度；关闭则回退默认下级配置。 |
| **角色插件映射** | 各角色默认可调用的代理插件列表（逗号分隔）。 |
| **任务限制** | 全局最大并发任务数；**代码任务**最大并发（建议常为 1）。 |
| **安全策略** | 是否允许代码任务、允许执行代码的插件白名单、是否拦截描述中的高危命令模式。 |

---

## 六、任务与调度

- 任务 **ID**、状态（pending / running / completed / failed / cancelled 等）。  
- **并发与角色容量**不足时任务 **入队**，并记录排队原因。  
- **资源锁**：避免特定资源冲突（实现于 `index.js`）。  
- **研究归档目录**：初始化时确保存档目录存在（`data/research-archive`）。

---

## 七、依赖与运行环境

- 插件目录下 **`npm install`** 可提供 `node-fetch`（无内置 `fetch` 时的 HTTP 兼容）。  
- 下级模型需支持 **工具/函数调用**。
